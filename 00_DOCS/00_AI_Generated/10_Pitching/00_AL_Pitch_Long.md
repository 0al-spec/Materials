# Agentification Layer — Unix для ИИ

⸻

## Слайд 1. Проблема: LLM‑хаос в продакшене

### Тезис:
Сегодня каждый AI-инженер строит свою инфраструктуру: shell-обёртки, хаки, костыли для доступа к данным, утилитам и системным вызовам.
LLM‑агенты не умеют нормально работать с ОС.
Нет стандарта. Нет ядра. Нет безопасности.

### Фраза для проговаривания:

> «Мы в 2025‑м, а LLM‑агенты до сих пор не могут вызвать top, прочитать лог или передать данные в Kafka без костылей и ручного кода. Всё разбито. Всё нестандартизировано. Это словно до Unix — всё самописное.»

⸻

## Слайд 2. Решение: Agentification Layer

### Тезис:
Agentification Layer превращает любую системную сущность (файл, утилиту, API, пайплайн) в стандартизированного агента, доступного через универсальный протокол (MCP/A2A).
Это ядро для любой AI‑инфраструктуры.

### Пример:
	•	Файл /etc/config.yaml → агент /get-config
	•	Скрипт train.sh → агент /run-training
	•	Kafka CLI → агент /publish
	•	top | grep PID → агент /get-metrics

### Фраза:

> «Мы превращаем всё — от ls до LLM‑ядра — в стандартизированные агенты. Без кода. Один YAML-файл — и любой LLM может вызвать его безопасно, через API.»

⸻

## Слайд 3. Как это работает

### Схема (в виде графа):

LLM agent → MCP/A2A → agentifyd → YAML passport → system call / CLI

### Тезис:
	•	LLM не знает, как читать файл — она вызывает /read-file.
	•	agentifyd исполняет это безопасно, с chroot, seccomp и cap‑листами.
	•	Всё как systemd + POSIX, но для AI.

### Фраза:

> «Мы создали POSIX‑слой для LLM. Вызов /run-preprocess из языка — это как syscall в ядро. Но не через libc, а через MCP.»
